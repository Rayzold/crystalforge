FIREBASE GM WRITE RULES

Use these Firestore rules when you want:
- everyone authenticated anonymously to READ the published realm
- only your GM browser UID to WRITE

STEPS
1. Open the app
2. Type 432!
3. Go to Admin Console -> System -> Firebase Realm
4. Click "Use Current Browser As GM Publisher"
5. Copy the shown GM publisher UID
6. Open Firebase -> Firestore -> Rules
7. Replace YOUR_GM_UID_HERE below with your real UID
8. Publish the rules

RULES

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /realms/{realmId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null
        && request.auth.uid == "YOUR_GM_UID_HERE";
    }
  }
}

NOTES
- This protects Firebase itself, not just the app UI.
- If you ever change GM browser/account, update the UID in both:
  - the app's GM publisher setting
  - Firestore rules
